# StaticDynamic v0.2.1

This release focuses on workflow hardening, report generation, and GUI layout
stability after the layered-soil `v0.2.0` release.

## Highlights

- Added preflight checks for function dependencies, model inputs, geostatic
  file/source consistency, spring-damping depth, and balance tolerance.
- Added `StaticDynamic_run_report.txt` and `StaticDynamic_run_report.csv`.
- Added boundary, material, geostatic reaction, spring-dashpot, load, step, and
  job summary statistics to each run report.
- Updated the GUI screenshot used in the README.
- Compacted the lower-left GUI layout so the wave-file selector remains visible
  in high-DPI Abaqus/CAE dialogs.
- Improved ODB/CSV geostatic file browser filtering and keyword synchronization.
- Added `examples/validate_current_session.py` for lightweight Abaqus-session
  boundary and tributary-weight validation.

## Validation Notes

Checked in Abaqus/CAE session with:

- homogeneous 3D soil box: 273 unique boundary nodes, bottom tributary area
  `1600`, and each side tributary area `1200`
- two layered 3D validation models with five active artificial boundary faces
- local Python syntax checks for the plugin modules and validation script
