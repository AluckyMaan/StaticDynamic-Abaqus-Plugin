# StaticDynamic v0.5.0

This release adds the first small earthquake-input workbench layer on top of
the v0.4.0 traveling-wave and layered-site workflow.

## Highlights

- Added `Wave Scale` to scale loaded acceleration, velocity, and displacement
  records after unit conversion.
- Added `Baseline Correction = RemoveMean` to subtract the mean from loaded
  motion records before equivalent-load generation.
- Added run-report audit fields for removed mean, scale, and peak values
  before/after preprocessing.
- Added `Site Profile CSV` for `Input Mode = LayeredSite`.
- Imported site profiles can provide `verticalCoordinate`, `equivalentVs`,
  optional `cumulativeTravelTime`, and optional `sampleCount`.
- If cumulative travel time is omitted, the plugin computes it from the sorted
  coordinate and `Vs` profile using trapezoidal slowness.

## Notes

`Site Profile CSV` gives users a clearer bridge from external site-response or
site-characterization tools into StaticDynamic. It is still a 1D kinematic
arrival-time input; it does not yet perform equivalent-linear amplification,
deconvolution, or free-field column coupling inside the plugin.

`Wave Scale` and `RemoveMean` are intentionally conservative preprocessing
steps. The plugin records the audit values in `StaticDynamic_run_report.csv`
and `StaticDynamic_run_report.txt` so the applied record can be checked before
using the final dynamic model.
