# StaticDynamic v0.6.2

This patch release focuses on code hardening and small performance improvements
before larger `0.6.x` refactoring work. It keeps the `v0.6.0` oblique-incidence
workflow and the `v0.6.1` validation fixes intact.

## Highlights

- Added shared CSV read/write helpers that use Python 3 text `newline=''` mode
  while preserving Abaqus Python 2 binary CSV behavior.
- Applied the CSV helpers to run reports, `NodeInfo.csv`, boundary audit CSVs,
  seismic arrival/profile CSVs, and geostatic reaction CSV input.
- Cached boundary-node coordinates during artificial-boundary detection and
  computed axis min/max values in one pass.
- Extracted shared coordinate clustering for layer-center grouping and
  tributary-width calculation.
- Hardened section material lookup for section objects that expose a
  `materials` collection instead of a single `material` attribute.
- Reconnected dialog `processUpdates()` to refresh the lightweight DB cache and
  option enabled/disabled states.
- Updated the Abaqus/CAE dialog and plugin menu to `StaticDynamic v0.6.2`.

## Notes

The `Main()` orchestration function remains intentionally unchanged in this
release. Larger flow extraction, ODB single-open reaction/balance reads, and
output-directory configurability are tracked for later `0.6.x` work so this
patch can stay easy to review and validate.

## Validation

The release was checked with:

```text
python -m py_compile StaticDynamic.py staticDynamicDB.py staticDynamic_Form.py staticDynamic_plugin.py examples\validate_current_session.py
```

Full Abaqus/CAE workflow validation should continue to use the `v0.6.0`
five-layer oblique-incidence benchmark as the reference scenario.
