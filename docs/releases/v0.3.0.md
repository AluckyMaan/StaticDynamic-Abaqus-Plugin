# StaticDynamic v0.3.0

This release adds practical earthquake-motion input for viscous-spring
boundaries.

## Highlights

- Read PEER `.AT2`, `.VT2`, and `.DT2` strong-motion files.
- Convert PEER acceleration (`g`), velocity (`cm/s`), and displacement (`cm`)
  into the selected model length unit (`m`, `cm`, or `mm`).
- Automatically load matching PEER companion files with the same base name.
- Create equivalent seismic boundary loads for each spring-dashpot group using:

```text
F(t) = K_node * u_g(t) + C_node * v_g(t)
```

- Generate missing velocity or displacement records by trapezoidal integration
  when only acceleration or velocity is available.
- Add GUI controls for `Wave Format = PEER` and `Model Length Unit`.
- Record seismic input files, units, scales, points, and created load counts in
  `StaticDynamic_run_report.txt` and `.csv`.

## Notes

The current seismic input is a direct uniform equivalent boundary input. It does
not yet perform layered free-field deconvolution, oblique wave scattering, or
spatially variable arrival-time correction.

## Next

The next planned milestone is `v0.4.0`, focused on earthquake-input
improvements: incident-wave direction, spatial arrival-time delay on artificial
boundaries, and site-response/free-field preprocessing before applying PEER
motions to the model.
