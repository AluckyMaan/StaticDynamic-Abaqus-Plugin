# StaticDynamic v0.4.0

This release stabilizes the first incident-wave input workflow for
viscous-spring artificial boundaries.

## Highlights

- Added `Input Mode = Traveling` for spatial arrival-time delay on boundary
  nodes.
- Added separate `Incident Vector` and `Propagation Vector` controls.
- Added `Apparent Velocity` and `Delay Bin Size` for arrival-delay grouping.
- Added `Input Mode = LayeredSite`, which derives a vertical `Vs` profile from
  model materials and applies depth-dependent arrival delays.
- Exported `SeismicArrivalInfo.csv` for node-level arrival-delay audit.
- Exported `SeismicSiteProfile.csv` for layered-site profile audit.
- Added report fields for arrival mode, propagation vector, apparent velocity,
  delay range, delay bins, site profile points, and per-face delay statistics.
- Added preflight checks for nonzero incident/propagation vectors, positive
  traveling-wave velocity, and excessive delay-bin counts.

## Notes

`Traveling` and `LayeredSite` are kinematic arrival-time corrections. They make
the boundary input more transparent and auditable, but they do not yet replace a
full free-field column calculation, deconvolution workflow, equivalent-linear
site response, or complete oblique-wave scattering solution.
