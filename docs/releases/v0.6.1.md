# StaticDynamic v0.6.1

This patch release merges the `0.5.1` code-review fixes into the `0.6.x`
oblique-incidence branch. It preserves the `v0.6.0` arbitrary-angle incident
wave workflow while bringing forward the hardening fixes from the `0.5.1`
branch.

## Highlights

- Kept `Incident Angle` and `Azimuth` as the primary `0.6.x` convenience path
  for traveling-wave propagation-vector generation.
- Fixed `Input Mode` normalization so hyphenated, spaced, or underscored
  traveling-wave aliases are handled consistently.
- Sorted arrival-delay bins numerically instead of lexicographically in global
  and per-face report statistics.
- Hardened site-profile CSV header detection so scientific notation values do
  not produce false header matches.
- Simplified `SeismicArrivalInfo.csv` export by removing an unused argument.
- Reused the shared vector-magnitude helper in unit-vector normalization.
- Updated the Abaqus/CAE dialog and plugin menu to `StaticDynamic v0.6.1`.

## Compatibility Note

The original `0.5.1` review branch required `Propagation Vector` to be filled
for traveling-wave input. In `0.6.x`, a blank `Propagation Vector` is valid
because the plugin can generate it from `Incident Angle` and `Azimuth`. This
release keeps the `0.6.x` behavior and only validates an explicit propagation
vector when one is provided.

## Validation

The release was checked with:

```text
python -m py_compile StaticDynamic.py staticDynamicDB.py staticDynamic_Form.py staticDynamic_plugin.py examples\validate_current_session.py
```

The full Abaqus/CAE workflow validation from `v0.6.0` remains the reference
oblique-incidence benchmark:

- five-layer `30 m x 20 m x 20 m` soil model
- 1 m C3D8R mesh
- geostatic reaction extraction
- viscous-spring boundary conversion
- 32-degree oblique traveling-wave earthquake input
- dynamic ODB field outputs `U`, `V`, `A`, and `RF`

See [StaticDynamic v0.6.0](v0.6.0.md) for the Abaqus screenshots and detailed
run counts.
